# 校享云 — 前端架构设计说明书

> 版本：v1.0 | 日期：2026-07-11 | 技术栈：Vue3 + TypeScript + Vite

---

## 一、项目定位

三端合一的高校后勤服务平台前端：

| 端 | 用户 | 载体 | UI 风格 |
|----|------|------|--------|
| **学生端 (C)** | 在校学生 | H5/小程序 | 年轻化、活泼、大图卡片、一键操作 |
| **师傅端 (B)** | 服务师傅 | H5/App | 简洁、信息密度高、收入可视化 |
| **运营端 (Admin)** | 学校运营/超管 | PC Web | 专业、数据驱动、深色侧栏、仪表盘 |

---

## 二、技术选型

| 类别 | 选型 | 理由 |
|------|------|------|
| 框架 | Vue 3.5 + Composition API | 最新生态，`<script setup>` 语法简洁 |
| 语言 | TypeScript 5.x | 类型安全，大型项目必备 |
| 构建 | Vite 6 | 秒级 HMR，比 Webpack 快一个数量级 |
| 路由 | Vue Router 4 | 官方路由，支持嵌套/守卫/懒加载 |
| 状态管理 | Pinia 2 | Vue3 官方推荐，API 比 Vuex 简洁 |
| HTTP | Axios + 拦截器 | JWT 自动续期、统一错误处理 |
| UI 框架 | Element Plus 2.9（Admin）/ Vant 4（H5） | 国内最成熟的 Vue3 组件库 |
| CSS 方案 | UnoCSS（原子化）+ CSS Variables | 按需生成，主题切换零成本 |
| 图表 | ECharts 5（Admin） | 数据大盘、趋势分析 |
| WebSocket | 原生 WebSocket + 心跳重连 | IM 即时通讯 |
| 地图 | 腾讯地图/高德地图 | 服务地址选择（H5） |
| 国际化 | vue-i18n | 预留多语言（中/英） |
| PWA | vite-plugin-pwa | H5 可安装到桌面，离线缓存 |
| 代码规范 | ESLint + Prettier + Husky | Git 提交前自动格式化 |

## 五、设计语言 & 视觉体系

### 5.1 设计哲学

> **"轻盈、可信赖、有温度"**

不是冷冰冰的后台管理系统，而是一个有人情味的校园生活平台。

### 5.2 色彩体系

| 用途 | 色值 | 说明 |
|------|------|------|
| 品牌主色 | `#4F6EF7` 校享蓝 | 年轻、科技感、可信任 |
| 品牌辅色 | `#FF6B6B` 珊瑚红 | 抢券/促销按钮、Badge |
| 成功 | `#22C55E` | 支付成功、完成 |
| 警告 | `#F59E0B` | 待处理、审核中 |
| 中性灰 | `#64748B` | 次要文字、边框 |
| 背景 | `#F8FAFC`（浅）/`#0F172A`（深） | 浅色/深色模式 |

### 5.3 设计特色

| 特色 | 说明 |
|------|------|
| **玻璃态卡片** | Admin 统计卡片用 `backdrop-blur` + 半透明背景，悬浮微阴影 |
| **渐变按钮** | 主操作按钮用 `#4F6EF7 → #7C3AED` 渐变，hover 上浮 |
| **微交互动画** | 页面切换滑入、数字滚动递增、骨架屏加载、Tab 下划线跟随 |
| **暗色模式** | CSS Variables 一键切换，深夜运维不刺眼 |
| **移动端手势** | H5 支持下拉刷新、左滑删除、长按多选 |
| **空状态插画** | 无数据时显示手绘风插画 + 引导文案，不死板 |

## 七、关键交互细节

| 场景 | 交互 |
|------|------|
| 抢券 | 按钮点击后 0.3s 内禁用 + 脉冲动画 + 结果弹窗（成功撒花/已抢光沮丧） |
| 下单 | 底部固定价格栏 + 滑动选择规格 + 确认动画飞入购物车 |
| 支付 | 倒计时 15 分钟 + 支付方式切换 + 成功后跳转订单详情 |
| 下拉刷新 | 顶部出现校享云 Logo 下拉动画，松手刷新 |
| 骨架屏 | 列表加载时显示灰色占位块 + shimmer 流光效果 |
| 空状态 | 可爱插画 + "还没有订单，去逛逛吧~" + 引导按钮 |
| 深色模式 | 点击右上角切换，CSS Variables 全局切换，0.3s 过渡 |
| Token 过期 | 响应 401 → 自动调 refreshToken → 重发原请求 → 用户无感 |

## 八、与后端接口对接

| 后端服务 | 前端调用方式 |
|---------|------------|
| 所有 REST API | Axios 统一封装，baseURL = `/api/v1`，代理到 Gateway `:8080` |
| JWT 认证 | 登录后存 `accessToken` + `refreshToken` 到 Pinia + localStorage |
| 文件上传 | 前端直传 MinIO（预签名 URL），不经过后端 |
| WebSocket IM | 原生 WebSocket 连接 `ws://localhost:8090/ws`，心跳 30s |
| 实时数据推送 | SSE 或 MQ → WebSocket 推送到前端（Dashboard 实时更新） |
